# v0.8 Release Validation

Arcane EDR `v0.8.x` is the GUI/MSI productization track. This checklist is the
release validation record for the MSI-owned local application model.

## Automated Local Checks

Run from the repository root:

```powershell
.\scripts\test-gui-shell.cmd
.\scripts\test-gui-scroll.cmd
.\scripts\test-gui-policy.cmd
.\scripts\test-gui-alerts.cmd
.\scripts\test-gui-payload.cmd -Path .\artifacts\gui\win-x64
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\test-fixtures.ps1
.\scripts\build.cmd
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-gui.ps1
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\package-release.ps1
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-msi.ps1
.\scripts\test-msi-validation.cmd -RunAdminValidation
```

For elevated repair validation, run from an elevated shell:

```powershell
.\scripts\test-msi-validation.cmd -RunSilentRepair -RunAdminValidation
```

The validation script checks MSI/ZIP presence, SHA256 checksum files, installed
service/GUI paths, installed service state, GUI payload coherence, and optional
admin validation or silent repair. GUI payload coherence means the deployed
`ArcaneEDR.Gui.dll`, `ArcaneEDR.Gui.pri`, and generated `*.xbf` resources are
present and from the same build window. This prevents stale WinUI XAML
resources from being left beside newer code. The script also reports
code-signing status. Unsigned public releases remain a hardening gate unless
the operator intentionally accepts unsigned artifacts.

## Structured CLI Seams

The GUI should prefer structured command output when available and fall back to
human text during upgrade windows. Current structured read-only outputs:

- `ArcaneEDR.exe --validate-config --json`
- `ArcaneEDR.exe --health --json`
- `ArcaneEDR.exe --alert-volume --last <duration> --json`
- `ArcaneEDR.exe --agent-activity --last <duration> --json`
- `ArcaneEDR.exe --incidents --last <duration> --json`
- `ArcaneEDR.exe --response-firewall list --json`
- `ArcaneEDR.exe --policy-inspect --json`
- `ArcaneEDR.exe --preview-daily-report --json`
- `ArcaneEDR.exe --preview-daily-report --ai-payload`

These outputs are local operator data. They may summarize local security
evidence and should be treated with the same care as logs and support bundles.

## Disposable-VM Gates

The following checks require a clean disposable VM or snapshot. Do not fake
these results from a live daily-driver workstation:

- clean install from release MSI
- upgrade from the previous public MSI
- repair after deleting an installed binary
- uninstall preserving config/logs
- explicit purge behavior if the tested installer exposes purge controls
- failed install rollback
- failed upgrade rollback
- silent install and silent uninstall
- high DPI, high contrast, keyboard-only, and Narrator GUI pass

Record the VM image, Windows build, install command, MSI version, result, and
log path for each run.
